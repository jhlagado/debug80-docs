# Chapter 11 — Reserving Storage with `.ds`

This chapter will explain how AZM reserves storage without confusing reservation with initialization. It will also introduce typed `.ds` forms and the named-count idiom for arrays.

## Planned sections

- Raw byte reservation
- Optional fill values
- Labels on storage
- `.ds byte[16]` and `.ds word[8]`
- Literal type-array counts
- Symbolic counts with `Count * sizeof(Type)`
- Storage maps
