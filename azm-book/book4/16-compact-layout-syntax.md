# Chapter 16 — Compact Layout Access Syntax

This chapter will introduce AZM's compact layout notation after the explicit `sizeof` and `offset` forms are understood. The emphasis is constant-address notation, not hidden runtime indexing.

## Planned sections

- Explicit `sizeof` and `offset` first
- Dot notation for nested fields
- Bracket notation for literal array elements
- Layout casts as constant-address notation
- Compile-time indexing vs runtime indexing
- Common mistakes
