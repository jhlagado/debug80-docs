# Using Debug80 in VS Code

Debug80 lets you run Z80 assembly programs inside VS Code, stop them at breakpoints, step through source lines, inspect registers and memory, and use panels that emulate machines such as the TEC-1 and TEC-1G.

This manual is for Z80 hobbyists. You may already know assembly, monitors, ROMs, or serial workflows. The manual focuses on what to click, what files Debug80 expects, what the generated files mean, and how to recover when VS Code does not launch the target you expected.
